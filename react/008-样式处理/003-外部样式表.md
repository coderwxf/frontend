```jsx
import './style.css'

export default function App(props) {
  return (
    <div>
      <h1 className='app-h1-title'>Hello React</h1>
      <h2 className='app-h2-title'>Hello World</h2>
    </div>
  )
}
```

```css
.app-h1-title {
  font-size: 20px;
  color: skyblue;
}

.app-h2-title {
  font-size: 18px;
  color: red;
}
```



`优点`

1. 结构样式分离
2. 样式文件可以被浏览器缓存
3. 可以使用CSS所有功能： 此方法允许我们使用CSS的任何语法，包括伪类、媒体查询等
4. 存在代码提示



`缺点`

1. 样式作用域是全局的
   + 需要人为约定样式命名规则，避免样式冲突
   + 使用css预处理器的样式嵌套功能来编写
     + 保证最外层样式名是唯一的
     + 组件中其余样式都是最外层样式的子样式
     + 从而避免样式冲突，但相应的预编译语言的嵌套，可能带来的就是超长的选择器前缀，从而降低应用性能
2. 样式中无法使用动态值
