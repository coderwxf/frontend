## 基本使用

一个类满足如下条件才可以被称之为`类组件`

1. 继承自 `React.Component` 或 `React.PureComponent`
2. 类必须实现 render方法，且render方法必须返回JSX元素 或 null 「也就是需要渲染的视图」
3. render方法必须位于类的实例的原型对象上

```jsx
// 这是ESM模块导入语法，不是解构操作
import { PureComponent } from 'react'

export default class App extends PureComponent {
  render() {
    return (
      <>
        this is a class component
      </>
    )
  }
}
```



## 特点

`Component` 和 `PureComponent`基本伪代码如下

```js
function Component (props, context, updater) {
  this.props = props;
  this.context = context;
  this.refs = emptyObject; // emptyObject 等价于 {}
  this.updater = updater || ReactNoopUpdateQueue;
｝
```



1. 类组件实例都继承了 `props`, `context` 和 `refs` 这三个实例属性 以及 `updater`实例方法

```js
class App extends Component {
  // 初始化 props，context以及updater
  constructor(props, context, updater) {
    // updater 多用于react内部，很少显示传入
    super(props, context, updater)
  }
}
```



2. `React.Component` 和 `React.PureComponent` 的原型对象上 存在

   + 属性 isReactComponent
   + 方法 forcelpdate
   + 方法 setState

   所以所有类组件都可以使用这两个方法 和 这一个属性
   
   这两个方法和一个属性 是被设置到组件实例的原型上的，不是实例属性
