## 语言分类

**计算机语言**: 

计算机语言是用来与计算机进行交流的工具，可以用来给出指令、执行指令并获得反馈。

计算机语言包括编程语言、标记语言、样式语言等。

不过标记语言和样式语言不属于编程语言的范畴，它们主要用于数据描述和显示样式，而不是编写逻辑来控制程序的流程。



**编程语言**

编程语言包括**低级语言**和**高级语言**。

- **低级语言**：如汇编语言，接近机器语言，主要用于硬件层面的编程。
- **高级语言**：如Python、Java、C++，更接近人类语言，易于理解和编写。

编程语言通常具有以下特点：

- **数据和数据结构**：能够处理基本数据类型和复杂的数据结构（如数组、列表、树等）。
- **指令和流程控制**：支持顺序执行、条件判断、循环等流程控制。
- **引用机制和重用机制**：支持通过函数、类、模块等方式实现代码的复用。
- **编程哲学和设计思想**：每种编程语言都有其独特的设计理念，例如Python强调简洁和可读性，Java注重面向对象编程。



## 语言发展

**机器语言**：

- 机器语言由0和1组成，也就是二进制编码，称为机器码。
- 它可以直接被处理器（CPU）执行，因此执行效率最高。
- 缺点是**可读性差**，编写过程繁琐且容易出错，同时也与具体的硬件架构紧密相关，**无法跨平台**。

**汇编语言**：

- 汇编语言使用助记符（如`MOV`、`ADD`等）替代机器码中的0和1，使得人类可以稍微容易理解。
- 汇编语言与机器语言类似，仍然与特定的硬件架构相关，**无法跨平台**。
- 尽管它的执行效率仍然很高，但与机器语言一样，**编写复杂**，并且仍需要程序员了解底层硬件。

**高级语言**：

- 高级语言更接近自然语言和人类思维模式，使得程序编写更加容易和直观。
- 高级语言通过**编译器**或**解释器**将代码转换为机器可以理解的二进制代码。虽然这个转换过程可能导致某些性能损耗，但现代编译器和解释器的优化已经能将性能影响降到较低水平。
- 高级语言的代码通常是**跨平台的**，因为它不依赖于特定的硬件架构。但某些语言需要通过不同的平台编译器生成对应的可执行文件。
- 各种高级语言（如Python、C++、Java等）通常为不同的应用领域和编程需求设计，导致了多样化，**无法完全统一**。



## JavaScript

## 编写位置

1. HTML元素上
2. script标签中
   + 可以存在多个script标签，他们作用域是互通的
   + 推荐script标签作为body标签中最后一个子元素 「 正常操作DOM 」
3. 外部JavaScript文件
   + 通过src属性引入外部脚本
   + 引入外部脚本后，script标签内部编写的js代码会被忽略

```html
<!-- 1. 自定义JavaScript代码优先于默认行为执行  -->
<!-- 2. html不区分大小写，所以onxxx事件是全小写的，不是小驼峰 -->
<a href="https://www.google.com" onclick="alert('Hello World')">click me</a>

<!-- href执行JavaScript代码，需要加上javascript: 前缀 -->
<a href="javascript: alert('Hello World')">click me</a>
```



## noscript

```html
<!-- 当支持js的时候，会执行script元素中的内容，不执行noscript中的内容 -->
<script>
  alert('Hello World')
</script>

<!-- 浏览器不支持JS代码的时候，会自动执行noscript元素中的内容  -->
<noscript>
  <p>浏览器不支持JS代码或已关闭了JS功能</p>
</noscript>
```



## 注意事项

1. JavaScript 严格区分大小写
2. script元素不推荐写成单标签 「 单标签不被推荐，且必须严格闭合 」
3. script元素默认的`type`值是`text/javascript`，也可以是其它的「 例如`text/babel` 」



## 交互方式

| 交互方式       | 功能           | 说明                                                         |
| -------------- | -------------- | ------------------------------------------------------------ |
| alert          | 普通弹框       | 1. 参数只有一个                                              |
| console.log    | 打印查看       | 1. 参数可以是多个<br />2. 多个参数之间通过空格分割           |
| prompt         | 输入弹框       | 1. 参数只有一个<br />2. 用户输入内容会自动转字符串<br />3. 用户取消输入，获取结果是 null |
| document.write | 追加或替换内容 | 1. 参数是多个<br />2. 多个参数作为普通文本直接插入「 紧紧挨在一起，没有空格 」<br />3. 控制台输出`document.write`会开启一个新的输入流<br />4. 如果是新开输入流，则替换之前所有页面内容<br />5. 如果是同一个输入流，则继续追加内容 |



## 语句和分号

1. 一条完整的操作就是语句
2. 运行后有结果的就是表达式，可以放`return`后



语句以分号结尾，但分号可以省略

JavaScript引擎将换行符识别为分号，这种机制被称之为`ASI机制`



但当以下情况时，分号不可以省略

1. 多行代码位于一行 「 代码压缩后 」
2. 以`[`、`(`、`{` 开头，且之前行不以分号结尾

```js
console.log('Hello World');

(function(){
  console.log('Hello JavaScript')
})()
```

```js
console.log('Hello World')

;(function(){
  console.log('Hello JavaScript')
})()
```

```js
console.log('Hello World')

// 以 +、-、！开头，JavaScript引擎会将其转换为表达式，从而和之前行分割开来
// 这种方式会将IIFE结果进行类型转换，但是不使用
// 所以更推荐通过加分号的方式进行分割
+(function(){
  console.log('Hello JavaScript')
})()
```
