## 网页的三大组成部分

- **HTML** 定义了网页的结构和内容。「 什么地方需要渲染什么样的内容 」
- **CSS** 美化了网页的外观和布局。 「 样式和布局 」
- **JavaScript** 添加了交互逻辑，使网页能够响应用户的操作并具有动态功能。「 动效 和 交互 」

其中HTML 是 标记语言，CSS是样式语言，JavaScript是编程语言，它们都是计算机语言



**计算机语言**：

+ 人与计算机之间的沟通桥梁 是指示计算机完成对应功能的一种方式。
+ 通过这些语言，你可以告诉计算机你希望它做什么，并且计算机会按照你的指示去执行相应的操作。
+ 计算机语言有很多分类: 机器语言、汇编语言、样式语言、编程语言、标记语言和查询语言等



**编程语言**：计算机语言的一个子集，可以通过自己的特有语法编写业务逻辑去实现自定义功能



编程语言与标记语言的区别在于，编程语言具备以下特点：

1. **数据和数据结构**：编程语言能够定义和操作各种数据结构，如数组、列表和对象。
2. **指令和控制流程**：编程语言包含用于控制程序执行流程的语句，如条件判断（if-else）和循环（for、while）语句。
3. **引用和重用机制**：编程语言支持代码的引用和重用机制，如函数、方法和模块。
4. **设计哲学**：每种编程语言都有其独特的设计理念。如面向对象编程，面向函数编程等



## 编程语言的发展历程

编程语言的发展可以分为三个阶段：

### 第一阶段：机器语言

机器语言是直接用0和1编写的代码。这种语言可读性差，容易出错，无法跨平台。但执行效率高，因为它不需要经过编译或汇编。

### 第二阶段：汇编语言

汇编语言使用助记符代替0和1，增加了可读性，但不同的机器有不同的汇编语言和汇编器，缺少移植性。汇编语言通常用于操作系统内核、驱动程序和单片机等硬件相关的开发。

### 第三阶段：高级语言

高级语言更接近人类的自然语言，易于理解和使用。它们脱离了对硬件的直接操作，具备很强的移植性。常见的高级语言包括JavaScript、Java、Python等。但高级语言需要经过编译或解释才能被计算机执行，存在一定的性能损耗。并且高级语言种类繁多，不同应用需要学习不同的语言



## 定义

JavaScript是一种基于原型的头等函数的多范式语言，支持面向对象、指令式编程和函数式编程

JavaScript的缩写是JS。它是一种高级的编程语言，最初是解释型语言，但在V8引擎中也可以看作是编译型语言 「 JIT 」



## 组成

JavaScript由三部分组成：

1. **ECMAScript**：这是JavaScript的标准，描述了该语言应具备的语法和基本对象。
2. **DOM（文档对象模型）**：用于操作网页中的各种元素。
3. **BOM（浏览器对象模型）**：用于操作浏览器窗口。

## JavaScript引擎

高级语言必须经过转换成机器指令才能被CPU执行。

JavaScript代码最终需要被CPU执行，而CPU只认识机器语言，所以需要一个JavaScript引擎将高级的JavaScript代码翻译成机器指令

常见的JavaScript引擎有很多，例如：

1. **SpiderMonkey**：第一款JavaScript引擎，由Brendan Eich开发。
2. **Chakra**：微软为早期IE浏览器开发的JavaScript引擎。
3. **JavaScriptCore**：由苹果公司开发，用于WebKit内核，常用于移动端浏览器和移动应用，以及小程序。
4. **V8引擎**：由谷歌开发，用于Chrome浏览器和Node.js，是目前最流行的JavaScript引擎。



### 浏览器内核和JavaScript引擎的关系

浏览器内核和JavaScript引擎的关系是：浏览器内核由两部分组成，一部分负责解析HTML和CSS，另一部分是JavaScript引擎

例如，WebKit内核由WebCore和JavaScriptCore组成，WebCore负责HTML和CSS的解析和渲染，JavaScriptCore负责JavaScript代码的解析和执行

在小程序中，渲染层（对应HTML和CSS）通过WebView 来渲染，逻辑层（对应JavaScript）通过JavaScriptCore来运行。



## 编写位置

1. **行内JavaScript代码**：简单直接，但不推荐使用。
2. **`<script>`标签内部**：适用于少量代码，易于管理。
3. **外部JS文件**：最推荐的方式，适用于大量代码，便于复用和维护



- 事件处理函数可以在 HTML 元素的属性中指定，事件名以 `on` 开头且全小写。
- 事件处理函数会在浏览器默认行为之前执行。如果需要阻止浏览器的默认行为，可以调用 `event.preventDefault()`。
- `return false` 会同时阻止浏览器的默认行为和事件冒泡，但更推荐使用 `event.preventDefault()` 和 `event.stopPropagation()` 来分别处理这两个行为。

```html
<a href="#" onclick="alert('百度一下')">百度一下</a>
```



在`a`元素的`href`属性中也可以编写简单的JavaScript逻辑，必须以`javascript:`开头

```html
<a href="javascript: alert('Hello World')">click me</a>
```





## noscript

如果浏览器不支持JavaScript，或者用户关闭了JavaScript功能，则执行`noscript`中的逻辑，否则就执行`script`中的逻辑



## 注意事项

- `<script>`元素不能写成单标签元素，即使引用外部JavaScript文件时也不能省略结束标签
- 如果引用了外部文件，不能在标签内再写JavaScript代码。
- `<script>`通过`type`指定脚本类型，默认值是`text/javascript`
- 为了确保JavaScript可以正确操作DOM，`<script>`标签最好放在`<body>`标签的最后一行
- JavaScript代码严格区分大小写



## 交互方式

   + 输出时以空格隔开

## ASI

通常情况下，每条语句结束时需要加上一个分号（semicolon）用于标识语句的结束

如果语句结束时紧跟一个换行符，分号是可以省略的。

JavaScript会将换行符理解为一个隐式的分号，这个过程称为自动插入分号（Automatic Semicolon Insertion）

但一行以`[,{,(`开头的时候，会导致ASI机制识别错误，将两行作为一行解析

解决方法如下(三选一):

1. 在当前行开头加上分号
2. 在前一行末尾加上分号
3. 在当前行开头加上`+,-,!` 「 将当前行识别为语句的开始，从而实现两行代码的解析分割 」



## 注释方式

1. **单行注释**：使用 `//`，只能注释一行。
2. **多行注释**：使用 `/* ... */`，可以注释多行。
3. **文档注释**：使用 `/** ... */`，通常用于函数或类的说明。



### 注意事项

1. 注释不能嵌套使用
2. 文档注释可以通过VS Code的插件自动生成
3. 在HTML文件中，VS Code可能不会提示文档注释的模板生成，因此建议在独立的JS文件中编写文档注释
